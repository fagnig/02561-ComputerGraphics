Reflection on affine transform theory

Matrices used in part 1:

A model-view matrix is constructed from a rotation-matrix (Angel 4.9.3), as no offset is required to show the wireframe cube in an isometric view. The matrix rotates the perspective 45 degrees around the x and y axis to achieve the isometric perspective. This matrix is created through eye-point positioning, which requires three three-dimensional vectors. A vector defining the point where the "camera" should be located, a vector defining what the camera should be looking at, and a vector defining which way is up for the camera. The matrix is constructed through the use of the lookAt() function (Angel 5.3.3).

The eye-point is calculated as a point on a sphere with a radius of 0.8.

The at-point is the origin (0,0,0).

The up-point is positive y (0,1,0).

The thing that varies when space is pressed is the eye-point, the angles used to calculate the point on the sphere is changed.

Matrices used in part 2:

A model-view matrix is again constructed through the eye-point positioning method.

The difference is, this time a perspective matrix is concatenated onto it.

A perspective projection matrix is set up using the perspective() function (Angel 1.6.2), taking a field of view, an aspect ratio and a near and far z position.
